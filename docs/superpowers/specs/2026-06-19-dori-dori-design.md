# Dori Dori — Game Design Spec

**Date:** 2026-06-19  
**Status:** Draft  
**Platform:** Web (PC + Mobile)  
**Stack:** Vite + TypeScript, MediaPipe Face Landmarker, PixiJS

---

## Overview

카메라 기반 목 스트레칭 슈팅 게임. 사용자의 얼굴이 화면 하단에 고정된 캐릭터 역할을 하고, 고개를 좌우로 기울여 발사 방향을 조준한다. 마법/환상 비주얼 컨셉.

---

## 1. 화면 구성 (Scenes)

### MainMenuScene
- 어두운 배경 + "DORI DORI" 타이틀 (중앙 상단, 큰 폰트)
- 버튼: 게임 시작 / 종료
- 앱 진입 시 카메라 권한 요청 + MediaPipe 백그라운드 초기화
- 초기화 완료 전: 시작 버튼에 로딩 스피너
- 카메라 권한 거부 시: 안내 메시지, 시작 버튼 비활성화
- 레퍼런스: 어두운 배경에 중앙 배치 타이틀 + 하단 메뉴 (Mutazione 스타일)

### GameScene
- 상단 HUD: 점수 / 콤보 카운터
- 하단 중앙: 카메라 얼굴 크롭 이미지 (원형, 고정 위치/크기, 캐릭터 역할)
- 적은 화면 상단(7시~5시 방향 호 범위)에서 등장해 하강
- 얼굴 미인식 시: 일시정지 + "얼굴이 인식되지 않습니다" 오버레이 → 재인식 후 1.5초 카운트다운 → 자동 재개
- 피버타임 시: 화면 테두리 황금빛 pulse 애니메이션

### GameOverScene
- 이번 점수 + 최고 점수 (localStorage)
- 버튼: 재도전 / 메인으로

---

## 2. 얼굴 추적 & 조준 메커니즘

- **라이브러리**: MediaPipe Face Landmarker (브라우저 WASM)
- 매 프레임 얼굴 인식 여부 + 머리 yaw(좌우 기울기) 각도 추출
- yaw 범위를 화면 X 발사 방향으로 매핑
  - 예: 고개 -30° (왼쪽) → 발사체 좌상방 발사, +30° (오른쪽) → 우상방
  - 정수리 좌표(얼굴 크롭 이미지 상단)가 발사 기점
- 얼굴 자체는 화면 고정, yaw 각도만 발사 방향에 반영

---

## 3. 발사체

- 자동 발사 (발사 간격: 600ms, 버프 시 250ms)
- 기본 발사체: 파란 마법 구체 (레퍼런스: 파랑/주황/초록 빛 궤적 이펙트)
- 피버타임 시: 발사체 이펙트 주황빛으로 전환
- 동시 최대 20개, 화면 밖 나가면 소멸

---

## 4. 적 (4종)

| 종류 | 체력 | 이동 | 크기 | 처치 점수 |
|------|------|------|------|-----------|
| 일반 | 1 | 직선 하강 | 소 | 10 |
| 강화 | 2 | 직선 하강 (느림) | 중 | 25 |
| 지그재그 | 1 | 좌우 진동하며 하강 | 소 | 20 |
| 보스형 | 3 | 직선 하강 (매우 느림) | 대 | 50 |

- 스폰 범위: 화면 상단 좌측~우측 (7시~5시 방향 호)
- 난이도 곡선: 스폰 간격 2000ms에서 시작, 시간 경과에 따라 최소 400ms까지 점진 감소
- 동시 화면 내 최대 15개

---

## 5. 아이템 (버프)

적 처치 시 낮은 확률(약 10~15%)로 아이템 드롭, 화면 하강.

| 아이템 | 효과 | 지속 |
|--------|------|------|
| 연사 속도 증가 | 발사 간격 600ms → 250ms | 10초 |
| 데미지 증가 | 발사체 데미지 +1 | 10초 |
| 발사체 크기 증가 | 충돌 반경 ×1.5 | 10초 |

- 버프 중복 불가, 새 아이템 획득 시 기존 버프 교체
- 아이템 획득 시 시각 이펙트 표시

---

## 6. 피버 시스템

- **트리거**: 미스 없이 적 10마리 연속 처치
- **지속**: 8초
- **효과**:
  - 점수 ×3
  - 화면 테두리 황금빛 pulse 애니메이션
  - 발사체 이펙트 주황빛 전환
- 피버 종료 후 콤보 카운터 리셋
- 콤보 리셋 조건: 적이 격파되지 않고 플레이어 아래로 화면을 통과(탈출)했을 때
- 적이 플레이어에게 닿으면 즉시 게임오버 (콤보 리셋 아님)

---

## 7. 점수 & 게임오버

- 점수: 적 처치 점수 합산, 피버타임 중 ×3
- 최고 점수 localStorage 저장
- 게임오버 조건: 적이 플레이어 얼굴 영역에 충돌
- 게임오버 화면: 이번 점수 / 최고 점수 / 재도전 / 메인으로 버튼

## 8. 아키텍처

```
dori-dori/
├── index.html
├── src/
│   ├── main.ts
│   ├── scenes/
│   │   ├── MainMenuScene.ts
│   │   ├── GameScene.ts
│   │   └── GameOverScene.ts
│   ├── tracking/
│   │   └── FaceTracker.ts       # MediaPipe 래핑, 얼굴 인식 여부 + yaw 각도 emit
│   ├── game/
│   │   ├── Player.ts            # 정수리 좌표 계산, 발사 로직
│   │   ├── Enemy.ts             # 적 베이스 + 4종 서브타입
│   │   ├── Projectile.ts
│   │   ├── Item.ts
│   │   ├── FeverSystem.ts       # 콤보 카운트, 피버 상태 관리
│   │   └── GameLoop.ts          # 스폰 타이머, 난이도 증가, 충돌 판정
│   └── ui/
│       └── HUD.ts               # 점수, 콤보, 오버레이
├── package.json
└── vite.config.ts
```

**렌더링**: PixiJS (2D WebGL) — 파티클/마법 이펙트, 모바일 성능 안정  
**빌드**: Vite + TypeScript

---

## 9. MVP 범위 제한

- 사운드/BGM 없음 (이후 추가)
- 온라인 리더보드 없음 (localStorage만)
- 배경 애니메이션 최소화
- 동시 화면 내 적 최대 15개, 발사체 최대 20개

---

## 10. 에러 처리 & 엣지케이스

- 카메라 권한 거부: 시작 버튼 비활성화 + 안내 메시지
- 얼굴 미인식 중: 게임 즉시 일시정지, 재인식 시 1.5초 후 자동 재개
- 모바일 화면 회전: 일시정지 후 비율 재계산 및 재개
- MediaPipe 로딩 중: 시작 버튼 로딩 스피너
